# Optimize Model Performance
### Exercise 1: Diagnosing Report Speed with the Performance Analyzer
**Objective:** Measure visual rendering times and extract underlying DAX queries using the Performance Analyzer to identify performance bottlenecks .
#### Data Input
Save the following code block as a CSV file named `Sales_Log.csv`.
**Code snippet:**
``` c
OrderID,CustomerName,SalesAmount,OrderDate
1,Alpha Corp,500,2023-01-01
2,Beta LLC,1200,2023-01-02
3,Alpha Corp,300,2023-01-03
4,Gamma Inc,850,2023-01-04
```
#### Step-by-Step Instructions
1. **Import Data:** Load `Sales_Log.csv` into Power BI Desktop.
2. **Construct Visual:** Switch to the **Report** view. Create a Matrix visual with `CustomerName` on Rows and `SalesAmount` in Values.
3. **Initiate Analyzer:** Navigate to the **View** ribbon. Select **Performance analyzer**.
4. **Record Interactions:** In the Performance Analyzer pane, click **Start recording** .
5. **Trigger DAX Query:** Interact with the Matrix visual by clicking the `CustomerName` column header to sort it .
6. **Analyze Metrics:** Observe the recorded metrics in the pane. Expand the entry for your Matrix. Review the execution time (in milliseconds) specifically allocated to the **DAX query** versus the **Visual display**.
7. **Extract DAX:** Click **Copy query** within the expanded metric details. Paste this query into a text editor to review the exact background DAX generated by the visual interaction .
### Exercise 2: Implementing and Syncing Slicers
**Objective:** Control visual filter contexts using distinct slicer configurations and synchronize filter states across multiple report pages .
#### Data Input
Save the following code block as a CSV file named `Inventory_Status.csv`.
**Code snippet:**
``` c
ProductID,Category,StockLevel,LastRestock
101,Furniture,50,2023-11-01
102,Technology,120,2023-11-05
103,Furniture,10,2023-10-20
```
#### Step-by-Step Instructions
1. **Import Data:** Load `Inventory_Status.csv` into Power BI Desktop.
2. **Create Slicer:** On Page 1, select the **Slicer** visual icon. Drag `Category` into the field well .
3. **Configure Slicer Format:** Because `Category` is text data, change its format. Click the caret (or access Format visual settings) and change the style from _List_ to _Drop-down_ .
4. **Enable Select All:** Navigate to the **Format visual** pane. Under **Slicer settings > Selection**, toggle **Show "Select all"** to _On_ .
5. **Replicate Slicer:** Right-click the slicer and select **Copy visual** (or use Ctrl+C) .
6. **Establish Synchronization:** Create a new page (Page 2). Paste the slicer using Ctrl+V. A **Sync visuals** dialog box will appear. Click **Sync** .
7. **Verify Sync State:** Select "Furniture" in the Page 2 slicer. Return to Page 1. The original slicer is now securely filtered to "Furniture".
### Exercise 3: Advanced and Top N Filtering via the Filters Pane
**Objective:** Apply multi-condition logic and isolate top performers using the Advanced and Top N configurations within the Filters pane .
#### Data Input
Use the `Sales_Log.csv` data from Exercise 1.
#### Step-by-Step Instructions
1. **Construct Baseline Visual:** On a new report page, create a Table visual containing `CustomerName` and `SalesAmount`.
2. **Apply Advanced Logic:** Expand the **Filters** pane. Drag `CustomerName` into the **Filters on this visual** well .
3. **Configure Advanced Filter:** Change the **Filter type** dropdown to _Advanced filtering_ . Configure the logic to: `contains "Corp" Or contains "Inc"`. Click **Apply filter** . The Table now excludes "Beta LLC".
4. **Apply Top N Logic:** Drag another instance of `CustomerName` into the **Filters on this visual** well. Change the **Filter type** to _Top N_ .
5. **Configure Top N Parameters:** Set **Show items** to _Top 1_. Drag the `SalesAmount` field from the Data pane into the **By value** data well. Click **Apply filter** . The Table dynamically resolves to show only "Gamma Inc".
### Exercise 4: Applying Themes and Canvas Formatting
**Objective:** Configure page-level aesthetics and apply customized report themes to enforce visual consistency .
#### Data Input
Ensure `Sales_Log.csv` (from Exercise 1) is loaded and a baseline Matrix visual exists on the canvas.
#### Step-by-Step Instructions
1. **Access Page Formatting:** Click the blank report canvas. Navigate to the **Format page** pane .
2. **Configure Wallpaper:** Expand **Wallpaper**. Change the Color to dark gray and set Transparency to 50%. This colors the entire report area .
3. **Configure Canvas Background:** Expand **Canvas background**. Change the Color to light blue. Set Transparency to 0% to explicitly distinguish the active visual layout area from the surrounding wallpaper .
4. **Apply Theme:** Navigate to the **View** ribbon. Expand the **Themes** gallery and select a pre-built theme to alter the global color palette .
5. **Customize Theme:** In the Themes dropdown, click **Customize current theme** .
6. **Modify Palette:** In the **Name and colors** section, alter _Color 1_ to a custom hexadecimal value. Click **Apply** . Observe the immediate color shift across all canvas visuals.
### Exercise 5: Accessibility Enforcement and Native Visual Controls
**Objective:** Implement strict accessibility standards and manipulate report focus using native visual controls .
#### Data Input
Use `Sales_Log.csv` from Exercise 1.
#### Step-by-Step Instructions
1. **Construct Visual:** Open the Report view. Create a Clustered Column Chart. Drag `CustomerName` to the X-axis and `SalesAmount` to the Y-axis .
2. **Enforce Accessibility Sizing:** Navigate to **Format visual > General** . Locate the text settings for titles and axes. Set all font sizes to a minimum of 15 points to meet baseline accessibility requirements.
3. **Configure Alternative Text:** Within the **General** formatting pane, locate the Alt Text field. Input a precise description of the visual's data. Do not use visual-dependent terms like "see" or "image of" .
4. **Execute Focus Mode:** Hover over the visual header. Click the **Focus mode** icon to isolate and expand the chart into a dedicated window, removing surrounding report distractions. Return to the main report.
5. **Execute Spotlight:** Click the **Ellipsis (...)** in the visual header. Select **Spotlight**. This forces user attention by retaining the visual's full opacity while fading all other elements on the canvas into the background.
6. **Export Visual Data:** Click the **Ellipsis (...)** again. Select **Export data** to extract the specific subset of data populating the visual into an external Excel or CSV file.
### Exercise 6: Filter State Control and Interaction Optimization
**Objective:** Restrict consumer filtering by locking and hiding filter cards, and optimize rendering performance by disabling cross-visual interactions .
#### Data Input
Use `Sales_Log.csv` from Exercise 1.
#### Step-by-Step Instructions
1. **Construct Baseline:** Create two Table visuals. Visual A contains `CustomerName` and `SalesAmount`. Visual B contains `OrderDate` and `SalesAmount`.
2. **Apply Global Filter:** Drag `OrderDate` to the **Filters on this page** well.
3. **Lock Filter State:** Click the **Lock filter** icon (padlock) on the filter card. This enforces the filter logic and prevents report consumers from modifying it .
4. **Hide Filter Card:** Click the **Hide filter** icon (eye) on the same card. This removes the card from the consumer-facing Filters pane, concealing the structural restriction.
5. **Enable Interaction Editing:** Select Visual A. Navigate to the **Format** ribbon and click **Edit interactions** to expose cross-filtering controls.
6. **Disable Cross-Filtering:** Locate the interaction icons appearing immediately above Visual B. Click the **None** icon (circle with a line) to strictly prohibit selections in Visual A from triggering queries or rendering updates in Visual B.
